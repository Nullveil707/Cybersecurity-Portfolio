# SQL Query Filtering for Security Investigations

## Project Overview

This project demonstrates the use of SQL queries to investigate security-related events and support administrative security tasks. By applying filtering techniques to organizational datasets, I was able to identify suspicious login activity, review authentication records, and locate employee systems that required security updates.

## Objectives

* Analyze login attempt data using SQL.
* Investigate failed and suspicious authentication attempts.
* Filter records based on dates, times, locations, and departments.
* Identify employee devices requiring security updates.
* Demonstrate the use of SQL as a cybersecurity investigation tool.

## Skills Demonstrated

* SQL querying
* Data filtering and analysis
* Cybersecurity investigations
* Security log review
* Database security concepts
* Problem solving

## SQL Concepts Used

* `SELECT`
* `WHERE`
* `AND`
* `OR`
* `NOT`
* `LIKE`

## Project Tasks

### 1. Investigated Failed Login Attempts

Retrieved unsuccessful login attempts that occurred after business hours to identify potentially suspicious activity.

### 2. Reviewed Login Activity by Date

Filtered login records from specific dates to support an investigation into unusual authentication events.

### 3. Identified Logins Outside Approved Locations

Used filtering techniques to locate login attempts originating outside designated geographic regions.

### 4. Retrieved Department-Specific Employee Records

Queried employee data to identify systems belonging to specific departments for security update deployment.

### 5. Identified Non-IT Employees Requiring Updates

Filtered employee records to locate users outside the Information Technology department.

This project strengthened my ability to use SQL to analyze security-related data and investigate potential cybersecurity incidents. It demonstrated how database queries can support threat detection, access monitoring, and security administration activities within an organization.
